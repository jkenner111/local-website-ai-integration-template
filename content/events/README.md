# content/events/

One MDX file per event. Filenames follow `YYYY-MM-DD-<slug>.mdx` — the date
prefix is what lets the file system stay chronologically ordered and lets
`lib/events.ts` find and parse events without a manifest.

See `2026-05-01-example-event.mdx` for the frontmatter shape. Delete that
file once you've added real events.
