An example event to show the event system's shape. Each event lives in its
own `.mdx` file under `content/events/`, named `YYYY-MM-DD-<slug>.mdx`.

The body (this paragraph) is rendered on the event's detail view. Frontmatter
drives the events list, the calendar, and the page metadata.

Supported frontmatter fields:

- `title` — required
- `date` — required, ISO (`YYYY-MM-DD`)
- `time` — optional, free-form display string
- `location` — optional
- `status` — `confirmed` | `tentative` | `canceled` (default: `confirmed`)
